# 手冊模板參考

這份參考檔用來幫你快速選擇合適的手冊結構。不要機械照抄，要先判斷：

1. 讀者角色是誰
2. 平台是什麼
3. 規模是小、中還是大

優先決定「讀者角色」，再決定「平台差異」，最後按規模擴充內容。

## 一、讀者角色模板

### 1. 一般使用者

適用情境：

- 新使用者第一次接觸產品
- 需要快速完成核心任務
- 不需要理解系統內部細節

推薦章節：

1. 這份手冊能幫你做什麼
2. 開始前準備
3. 第一次進入系統
4. 完成第一個核心任務
5. 常用功能
6. 常見問題
7. 附錄與檢查清單

撰寫重點：

- 先讓讀者成功，再講進階功能
- 用任務語言，不要用模組清單語言
- 每個步驟都說明預期結果
- 補充「遇到問題先看哪裡」

### 2. 管理員 / 後台操作人員

適用情境：

- 管理員、營運、內容審核、資料維護
- 需要操作表單、查詢、篩選、匯入匯出

推薦章節：

1. 權限與登入前提
2. 首頁與導航
3. 查詢 / 篩選 / 檢視明細
4. 新增 / 編輯 / 批次操作
5. 匯出 / 匯入
6. 風險提醒與復原方式
7. 附錄與檢查清單

撰寫重點：

- 釐清每個欄位或按鈕的用途
- 特別說明不可逆操作
- 補上常見誤操作與恢復方法
- 對批次操作與權限差異寫清楚

### 3. 維護者 / 值班人員

適用情境：

- 維運、值班、排查、現場支援
- 需要快速定位畫面、設定與錯誤徵兆

推薦章節：

1. 適用對象與系統邊界
2. 環境與權限確認
3. 核心流程驗證
4. 常見異常與排查路徑
5. 需要回報或升級處理的情況
6. 補充頁面與設定索引
7. 附錄與檢查清單

撰寫重點：

- 先寫「怎麼確認系統正常」
- 再寫「哪裡出錯會怎麼看」
- 補上頁面路徑、狀態提示與風險提醒
- 補一段最短排查順序

## 二、平台差異模板

### 1. Web

補充章節：

- 路由 / 導航入口
- 瀏覽器環境與權限
- 響應式差異

截圖重點：

- 入口頁
- 主工作區
- 導航 / 設定 / 重要彈窗

### 2. 桌面端

適用情境：

- Electron、原生桌面 App、桌面工具
- 涉及視窗、選單列、檔案操作、本機設定

補充章節：

1. 安裝與啟動
2. 主要工作區介紹
3. 核心任務流程
4. 檔案、匯入匯出與本機設定
5. 更新、同步與常見問題

截圖重點：

- 啟動畫面
- 主視窗
- 本機檔案操作區
- 設定與同步畫面

### 3. 行動端

適用情境：

- App、PWA、手機版 Web
- 同一功能在手機與桌面差異較大

補充章節：

1. 裝置與版本前提
2. 首次進入與導航方式
3. 主要任務流程
4. 手勢、底部導航或浮動按鈕說明
5. 常見卡點

截圖重點：

- 首頁首屏
- 漢堡選單 / 底部導航
- 任務完成前後的差異畫面

## 三、規模擴充方式

### 小規模

適合單一主流程。保留：

- 目標讀者
- 前置條件
- 主流程
- 2 到 4 張關鍵圖
- 常見問題
- 簡短附錄

### 中等規模

這是默認規模。除小規模內容外，再補：

- 重要分支流程
- 失敗場景
- 截圖圖說
- 已驗證 / 未驗證範圍
- 質量檢查清單

### 大規模

適合複雜產品或多角色。除中等規模內容外，再補：

- 角色差異
- 多入口或多模組關聯
- 更完整的排查與例外路徑
- 術語表或索引

## 四、圖說模板

每張圖至少補一段圖說，可直接套用這個結構：

`這張圖用來確認 [用途]。請先看 [關鍵區域]。與前一步相比，這裡的差異是 [狀態變化]。`

範例：

`這張圖用來確認聊天首頁是否正常載入。請先看右上角的新對話按鈕與下方輸入框。與剛進入頁面時相比，這裡已經顯示出完整對話區。`

## 五、章節順序判斷

如果你在「先功能導覽」還是「先任務步驟」之間猶豫，用這個判斷：

- 讀者是新手：先任務
- 讀者是維護者：先環境與排查
- 讀者是管理員：先權限與高頻操作
- 文件主要用途是培訓：先全局概覽，再進核心任務

## 六、不要這樣寫

- 不要只有畫面描述，沒有任務結果
- 不要只有流程，沒有失敗場景
- 不要只放截圖，不寫圖說
- 不要把未驗證的內容寫得像已確認
