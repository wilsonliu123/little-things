# 質量檢查清單

這份參考檔用來在交付前做最後把關。不要全部機械照抄，但至少要確保核心檢查項被覆蓋。

## 一、正文完整度

- 是否明確寫出文件目的與適用對象
- 是否交代這份手冊覆蓋哪些頁面、任務與角色
- 是否有清楚的主流程，而不是只有功能羅列
- 是否交代成功後應該看到什麼結果
- 是否補上常見問題、失敗場景或排查方向

## 二、截圖與圖說

- 截圖是否來自真實頁面操作
- 圖片檔名是否可讀且與內容一致
- 每張圖是否有圖說
- 圖說是否說清楚用途、關注區域與狀態變化
- 圖片順序是否與步驟順序一致
- 若有 HTML / PDF 導出，中間 HTML 是否真的含有 `<img>`
- 最終成品裡是否沒有殘留 `!圖片說明` 這種 Markdown 原文

## 三、真實性與邊界

- 是否避免捏造未驗證內容
- 是否清楚寫出已驗證範圍
- 是否清楚寫出未驗證範圍
- 若有假設，是否明確標示
- 若流程被阻塞，是否寫出阻塞原因

## 四、HTML / PDF 導出

- 若最終輸出 PDF，是否避免使用 emoji
- 中文 UI 名稱是否改用粗體，而不是放進反引號
- 若用了反引號，內容是否以 ASCII 路徑、URL、檔名或 ID 為主
- 導出前是否檢查過圖片路徑真的存在
- 是否抽查過 HTML / PDF 至少 2 到 3 個位置，確認截圖有出現

## 五、角色與平台匹配

- 一般使用者手冊是否聚焦任務完成
- 管理手冊是否講清楚權限、欄位與風險
- 維護手冊是否有最短排查順序
- 行動端手冊是否真的採用手機視角
- 桌面端手冊是否補上視窗、選單或本機操作差異

## 六、規模控制

### 小規模

- 是否聚焦單一主流程
- 是否避免塞入過多延伸章節

### 中等規模

- 是否包含主流程、常見問題、圖說、附錄與檢查清單
- 是否在完整與可讀之間取得平衡

### 大規模

- 是否真正需要多角色、多模組或大量排查內容
- 是否避免只是把中等規模內容無節制拉長

## 七、附錄建議

可按場景擇要加入：

- 驗證環境
- 截圖清單
- 頁面 / 路由索引
- 術語表
- 未驗證範圍
- 版本或日期資訊

## 八、交付前最後一眼

在交付前，快速問自己：

1. 這份文件真的是寫給指定讀者看的嗎？
2. 如果我是讀者，能不能跟著這份文件完成任務？
3. 如果卡住，我能不能從這份文件知道先看哪裡？
